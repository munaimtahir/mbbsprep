# Legacy and Historical Files

## Categories

### Active
Files that match current verified behavior and are still part of the working repository.

### Historical
Files that describe earlier repo states, prior audits, or earlier intentions.

### Legacy
Files, scripts, or notes that may still be informative but should not be assumed valid without direct verification.

## Rules for Historical Material

1. Historical files should not be deleted casually if they still provide context.
2. Historical files should be archived or clearly labeled.
3. Historical files must not override current code truth.
